## Pre-Push Checklist

Before pushing to GitHub:

1. âœ… **Test build script:**
   ```bash
   # Windows
   build-native.bat

   # Linux/macOS
   ./build-native.sh
   ```

2. âœ… **Verify `.gitignore` is working:**
   ```bash
   git status
   ```
   Should NOT show `android/`, `ios/`, or `node_modules/`

3. âœ… **Update README.md with:**
   - Your actual GitHub repository URL
   - Any additional setup instructions
   - Screenshots (optional but nice!)

4. âœ… **Check file permissions:**
   ```bash
   # Ensure shell script is executable
   chmod +x build-native.sh
   git add build-native.sh --chmod=+x
   ```

5. âœ… **Clean commit:**
   ```bash
   git add .
   git commit -m "Initial commit: PlayCast IPTV with HTTP Server & PiP support"
   ```

## After Cloning (For Users)

When someone clones your repo, they should:

1. **Clone:**
   ```bash
   git clone https://github.com/YOUR_USERNAME/PlayCast.git
   cd PlayCast
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Run build script:**
   ```bash
   # Windows
   build-native.bat

   # Linux/macOS
   chmod +x build-native.sh
   ./build-native.sh
   ```

4. **Select platform and build!**

The script will automatically:
- Eject Expo (creates `android/` and `ios/`)
- Copy native modules
- Configure build.gradle
- Register packages
- Build the app

## Quick Push Commands

```bash
# Initialize git (if not already)
git init

# Add remote
git remote add origin https://github.com/YOUR_USERNAME/PlayCast.git

# Check what will be committed
git status

# Add all files (respecting .gitignore)
git add .

# Commit
git commit -m "Initial commit: PlayCast IPTV Player

Features:
- IPTV player with M3U/M3U8 support
- HTTP Server for LAN file sharing
- Picture-in-Picture mode
- Advanced video controls
- Watch history and recommendations
- Multi-language support"

# Push to GitHub
git push -u origin main

# Or if using master branch
git push -u origin master
```
